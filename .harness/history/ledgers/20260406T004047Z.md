- **ResizeObserver + inline padding for gauge edge alignment**: Rather than hardcoding offsets or using a shared CSS grid, a `useLayoutEffect` in `App.tsx` attaches a `ResizeObserver` to `.weight-controls-row`, queries `:scope > .gauge-group` bounding rects, and derives `searchPadding.left/right` offsets passed as inline style to `SearchPanel`. This keeps the alignment live across resizes and degrades correctly (falls back to default 32px) when gauges wrap to multiple rows (`allSameRow` heuristic with 10px tolerance).

All four Browse rows (search, tabs, filter, table) now uniformly apply `var(--content-gutter)` (32px) as padding or margin with no branching. The `matches-table-wrapper` was split into its own standalone rule so the matches panel retained `overflow:auto; flex:1` without inheriting the browse-specific margins.

- **CSS-only `::after` tooltip on `data-tooltip` attribute**: Factor label tooltips are implemented as a pure CSS `::after` pseudo-element keyed to `data-tooltip`. Zero JS, zero new component, no layout impact. The `FACTOR_TOOLTIPS` map in `WeightControls.tsx` covers all 12 rendered factors. Tooltip text is suppressed (`?? undefined`) for factors without a defined entry.

- **`weightsLoading` as `useLayoutEffect` dependency**: The ResizeObserver setup re-runs when `weightsLoading` transitions, ensuring the observer is attached after the gauge row mounts. This correctly handles the initial load sequence without requiring a separate `useEffect` for observation lifecycle management.

## Durable Repo Guidance

- **ResizeObserver pattern for cross-component edge alignment**: When a child component's internal layout (gauge positions) must anchor a sibling's padding, measure via `ResizeObserver` in the shared parent and pass computed offsets as inline style props. Do not reach into child DOM from CSS alone for this kind of dynamic alignment.
- **CSS-only tooltips via `data-tooltip` + `::after`**: Suitable for low-interactivity label hints in this desktop-only context. Use `text-transform: none; letter-spacing: normal` to override label uppercase, and `pointer-events: none` to prevent hover interference. Not suitable if touch/mobile support is ever required.
- **`--content-gutter` as the single horizontal layout token**: All Browse rows should apply margins/padding via `var(--content-gutter)`. Any new Browse row must follow this pattern to maintain edge alignment without additional measurement changes.
- **`track-table-wrapper--framed` is fully removed**: No conditional framing logic exists. Do not reintroduce it — the unconditional margin approach replaces it.
