## Outcome

- Task: Fix 6 ingestion pipeline bugs + write tests + remediate 1 regression
- Result: PASS (score 100/A after remediation)

## Key Decisions

- Removed `ID3Tag.ENERGY` from shared `TAG_COLUMNS` rather than adding `energy` to all models. The database uses reflection and `initial_tags` has no `energy` column — the fix must match the actual schema.
- Added `PostMIKRecordFactory.update_row()` to read energy from `audio_file.parse_energy()`. This isolates energy to the MIK stage where it belongs.
- Changed `_build_final_row` to `return` the dict instead of `self.row = dict`. The original code assigned to `self.row` as a side effect, then `update_row` overwrote it with `None` (the return value). A subtle mutation-vs-return bug.
- Guarded `_get_final_bpm`/`_get_final_key` empty-dict cases with early `return None`. Choosing `None` over raising allows partial records and is consistent with nullable DB columns.
- After adding `None` returns to `_get_final_bpm`, `FinalPipelineStage.write_tags()` needed a corresponding `float(bpm) if bpm is not None` guard to avoid TypeError. Breaker caught this regression; fixed in remediation round 1.

## Verification Learnings

- The `float(tag_record.bpm)` crash in `write_tags()` was a latent assumption baked into the final stage: "final BPM is always set." The fix made that assumption false for edge cases, revealing the downstream dependency. Pattern: when a function changes from "always returns a value or raises" to "may return None", all callers of that value need auditing.
- Breaker correctly identified that fixing crash-earlier behavior with "return None" can move partial state into the DB. Worth checking for this pattern in future pipeline fixes.

## Breaker / Regression Learnings

- `_build_final_row` still accesses `initial_record.title` and `post_mik_record.energy` without None guards (pre-existing). Caught by `create_tag_record()` try/except, so no crash. Deferred as a follow-up NIT.
- Adding new `None` return paths should always trigger a search for `float(x)`, `int(x)`, `str.method(x)` callers of that value.

## Durable Repo Guidance

- `TAG_COLUMNS` in `ingestion_pipeline/config.py` is a shared base for ALL factory stages. Only add a column there if ALL stages' DB tables have that column. Stage-specific fields must be added in `update_row()` overrides.
- When `_get_final_bpm` or `_get_final_key` returns `None`, downstream write operations must guard before numeric conversions.
- The 4-stage ingestion pipeline uses the Template Method pattern (`update_row()` hook). New per-stage logic goes in `update_row()` overrides, never in the base `create_row()`.

## Deferred / Follow-up

- Optional: add None guards for `initial_record` and `post_mik_record` in `_build_final_row` (pre-existing risk, caught by try/except).
- Optional: consider whether `FinalTagRecord` with `bpm=None` is semantically correct or should be prevented upstream.
